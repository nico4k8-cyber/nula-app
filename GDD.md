# SHARIEL — Game Design Document (MVP)

**Статус:** MVP (Minimum Viable Product)
**Дата:** 2026-03-24
**Версия:** 1.0
**Аудитория:** Дети 7-14 лет

---

## 📋 Executive Summary

**SHARIEL** — это TRIZ-тренажёр, который учит детей видеть скрытые закономерности природы и применять методы решения проблем. Основано на курсе "Продюсирование игр" (Кирилл Золовкин) и педагогике "Обучения решателей" (Анатолий Гин).

**Основное постулирование:** Древние знания о том, как природа решает невозможные задачи, зашифрованы в её явлениях. Ребёнок учится их "расшифровывать" и применять в реальной жизни.

## 🎮 Game Loop (Core Gameplay)

### Сценарий игрока в одном цикле:

```
1. ВДОХНОВЕНИЕ (Onboarding)
   ├─ Дракон Орин показывает горящий мир с проблемами
   └─ "Есть способ всё это пофиксить. Я научу тебя"

2. ВЫБОР ЗАГАДКИ (Main Menu)
   ├─ Ребёнок видит 6 загадок о природе
   └─ Выбирает одну

3. ПАРАДОКС (Puzzle Screen)
   ├─ Вопрос: "Почему рыба-клоун живёт в ядовитых щупальцах и не гибнет?"
   ├─ Ребёнок читает вопрос
   └─ "Как это возможно?"

4. ИНТУИТИВНЫЕ ПОПЫТКИ (Thinking Phase)
   ├─ Ребёнок пишет свою идею: "Может, у неё иммунитет?"
   ├─ Система подтверждает / уточняет
   └─ "Хороший ход! Но давай посмотрим на природу..."

5. РАСКРЫТИЕ МЕТОДА (AI Dialogue)
   ├─ Орин объясняет: "Это называется ПОСРЕДНИК"
   ├─ Показывает животное-пример (рыба-клоун)
   └─ "Между рыбой и анемоной есть третий элемент — слизь. Благодаря ей они дружат"

6. ПРИМЕРЫ ИЗ ПРИРОДЫ (Discovery)
   ├─ Система показывает 3+ примера ПОСРЕДНИКА в природе
   ├─ - Пчела опыляет цветок (посредник: пыльца)
   ├─ - Рыба чистит зубы крокодилу (посредник: услуга)
   └─ - Люди связывают врагов через переговорщика

7. ПРИМЕНЕНИЕ В ЖИЗНИ (Achievement)
   ├─ "Теперь видишь, как метод работает везде?"
   ├─ Ребёнок открывает новое здание в городе (🏰 Замок Союза)
   └─ "Метод ПОСРЕДНИК теперь твой!"

8. ГОРОД МЕТОДОВ (Progress Visualization)
   ├─ Карта города вырастает
   ├─ Видно: 1 из 6 зданий открыто
   └─ Мотивация: "5 методов ещё ждут"
```

### Ключевые отличия от неправильного подхода:

| Неправильно | Правильно (TRIZ-педагогика) |
|--|--|
| Показать ответ → ребёнок заучивает | Парадокс → ребёнок сам ищет → метод |
| "Это называется ПОСРЕДНИК" | Ребёнок сам видит явление, потом узнаёт название |
| Одна загадка про разные методы | Одна загадка, один метод, много примеров из природы |
| Сразу дать решение | Дать интуитивное пространство для попыток |

## 🎯 Core Design Principles

### 1. **Парадокс первичен**
Ребёнок должен столкнуться с невозможным раньше, чем узнает метод. Это включает мозг.

### 2. **Природа — учитель**
Методы ТРИЗ — это то, как природа решала эти задачи миллионы лет. Ребёнок учится у природы, не у людей.

### 3. **Одна загадка = Один метод = Много примеров**
Не случайные загадки. Каждая загадка раскрывает ровно один метод через природные примеры.

### 4. **Прогресс видимый**
Город растёт. Ребёнок видит результат своего обучения. Это мотивирует повторное решение загадок.

### 5. **Атмосфера первична** (по Золовкину)
Звук, визуал, текст создают ощущение "я в приключении с драконом", а не "я выполняю задание".

## 🧠 Педагогическая механика (по Анатолию Гину)

### Две функции образования:
1. **Передача культуры** → Ребёнок узнает о методах ТРИЗ
2. **Формирование культуры изменений** → Ребёнок начинает думать как инженер/изобретатель

### Цикл обучения "Решателя":

```
Парадокс (удивление)
    ↓
Интуитивные попытки (ребёнок сам ищет, не видит ответ)
    ↓
Раскрытие принципа (Орин объясняет суть)
    ↓
Примеры из жизни / природы (генерализация)
    ↓
Успешное применение (ребёнок открывает новый метод)
    ↓
Готовность к новой задаче (мотивация)
```

### Роль Орина (дракона):
- Не даёт ответы, а задаёт вопросы
- Показывает примеры природы, а не людей
- Хвалит за попытки, даже если неправильно
- Помогает увидеть скрытую закономерность

## 📊 Прогресс и мотивация

### Визуальные сигналы:

1. **Progress Bar** — линия от 0% к 100% (6 методов)
2. **City Map** — растущий город (видимый результат)
3. **Achievements** — "Половина пути! Ты видишь, как думают инженеры"
4. **Stars** (отключены в МВП) — будущая система оценки

### Повторная игра:

Ребёнок может решить одну загадку много раз:
- Первый раз: открывает метод
- Второй раз: укрепляет понимание, видит город растущим
- Третий раз и далее: может получить разные вариации (future)

## 🎬 User Flows

### Flow 1: Первый раз (без сохранённого прогресса)

```
App loads
    ↓
[Check localStorage]
    ↓
No saved state
    ↓
Onboarding Screen (эпическое вступление)
    ↓
[User clicks "НАЧАТЬ"]
    ↓
Age Select (выбор возраста)
    ↓
[User selects age]
    ↓
Task Picker (список 6 загадок)
```

### Flow 2: Повторное открытие (есть прогресс)

```
App loads
    ↓
[Check localStorage]
    ↓
Has saved state + hasSeenOnboarding = true
    ↓
Skip Onboarding
    ↓
Jump to Task Picker (прямо на список загадок)
```

### Flow 3: Решение загадки

```
Task Picker
    ↓
[User selects puzzle]
    ↓
Puzzle Screen (показ вопроса)
    ↓
[User types answer]
    ↓
AI Dialogue with Orin (разбор ответа)
    ↓
Debrief (примеры метода)
    ↓
Outcome (открытие здания + поздравление)
    ↓
City Map (опционально — просмотр прогресса)
    ↓
Back to Task Picker
```

## 🎯 Success Criteria (для плейтестов)

### Что ожидаем увидеть у ребёнка:
1. **Вход:** "О, это про природу? Интересно..."
2. **Первая загадка:** "Как это работает? Я не знаю..."
3. **Попытка ответа:** "Может быть, потому что..."
4. **Раскрытие метода:** "Ах, это ПОСРЕДНИК! А я тогда видел это в..."
5. **Выход:** "Это так круто! Я хочу ещё!"

### Метрики успеха:
- ✅ Ребёнок понимает концепцию без объяснений
- ✅ Возвращается, чтобы решить ещё одну загадку
- ✅ Применяет метод к своей жизни ("Как в природе...")
- ✅ Рекомендует друзьям

### Метрики неудачи (красные флаги):
- ❌ Ребёнок запутался в интерфейсе (не понимает, что нажать)
- ❌ Потеря интереса через 2 минуты
- ❌ Вопрос: "Что это вообще такое?" (онбординг не сработал)
- ❌ Технические ошибки (крашится, зависает)
