## 🔌 API Endpoints

### GET /api/tasks/:taskId
Получить полную структуру задачи

```bash
curl http://localhost:3001/api/tasks/8
```

### POST /api/tasks/:taskId/validate-answer
Валидировать ответ ребенка

```bash
curl -X POST http://localhost:3001/api/tasks/8/validate-answer \
  -H "Content-Type: application/json" \
  -d '{
    "answer": "Убегать вместе в разные стороны",
    "userId": "user123",
    "sessionId": "session456",
    "source": "test"
  }'
```

Ответ:
```json
{
  "isCorrect": false,
  "score": 3,
  "earnsCrystal": true,
  "feedback": "✨ Отличное решение!",
  "chosenOption": { "text": "Убегать вместе в разные стороны", "temp": "bingo" },
  "realSolution": "Зебры используют несколько стратегий...",
  "bonusFact": "Корабли раскрашивают в полосы..."
}
```

## 🔄 Как это работает

1. **Frontend** (React/Vite):
   - Загружает задачи из локального TASKS массива (src/tasks.js)
   - Ребенок пишет решение
   - Отправляет ответ через API

2. **Backend** (Node.js/Express):
   - Получает ответ в валидации эндпойнте
   - Ищет substring matching в доступных опциях
   - Возвращает score (1-3 звезды) в зависимости от temp (cold/warm/bingo)
   - Успешные ответы дают кристалл

3. **Валидация**:
   - NLU-подход: substring matching игнорирует регистр
   - Все решения валидные, разница только в elegance
   - Score зависит от ТРИЗ принципа эффективности

## 🎮 Как ребенок играет

1. Видит задачу в UI
2. Придумывает решение (говорит в микрофон или пишет)
3. System валидирует через backend
4. Если найдено matching решение → ✓ + звездочка
5. Показывается real solution и bonus fact
6. Орин спрашивает "Хочешь найти еще способ?" → back to step 2
7. После 10 решений → переход на следующую задачу
