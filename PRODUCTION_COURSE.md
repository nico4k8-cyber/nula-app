# Module 1: Trainer Overview (30 minutes)

## What Problem Are We Solving?

**The Challenge:** How do we teach kids (6-12 years) to think creatively and solve problems systematically?

**Traditional Approach (Broken):**
- Textbooks: Boring, passive reading
- Teachers: Limited availability, scalability issues
- Tutors: Expensive, one-on-one only

**Our Solution:**
An AI-powered ТРИЗ coach that guides kids through solving real problems **without telling them the answer**.

## Why ТРИЗ?

**ТРИЗ** (Russian: "Theory of Inventive Problem Solving") is a systematic framework used by engineers and innovators worldwide:
- Not random brainstorming — structured thinking
- Proven to work with kids (develops metacognition)
- Creates confidence in problem-solving ability

## What Makes Our Trainer Different?

### Core Philosophy: "Ask, Don't Tell"

Traditional tutoring:
```
Child: "How do I solve this?"
Teacher: "Use method X..."
(Child memorizes, doesn't learn)
```

Our trainer:
```
Child: "How do I solve this?"
Уголёк: "What do you notice about this part?"
(Child thinks, discovers, learns)
```

### The ПРИЗ System: 5-Stage Scaffolding

We break problem-solving into 5 digestible stages:

| Stage | Letter | What Happens | Duration |
|-------|--------|-------------|----------|
| **Подготовка** | П | Engage: "What's really happening here?" | 1 message |
| **Разведка** | Р | Explore: "What resources do we have?" | 1-2 messages |
| **Идеи** | И | Generate: "How could we solve it?" | 1 message |
| **Зачёт** | З | Validate: "Does this work?" | 1 message |
| **Инсайт** | ✨ | Celebrate: "You solved it! Here's why..." | 1 message |

**Total:** 5-7 messages per task = 15-30 minute session (optimal engagement window per production course)

### Star System: Quality Over Correctness

Instead of "right/wrong," we reward *thinking quality*:

- ⭐ (1 star): Noticed a resource or asked a smart question
- ⭐⭐ (2 stars): Found a non-obvious solution path
- ⭐⭐⭐ (3 stars): Insight that flips the problem on its head

**Why?** Kids remember what they *thought about*, not what they were *told*.

## Demo: Playing Through One Task

**Task:** "A boy is in a room filled with oil. How does he cross to the door?"

### Stage П (Подготовка) - Engage
```
Уголёк: "Вот мальчик, вот дверь, а пол весь в масле. Скользко!
         Скажи мне — ты когда-то скользил? На чём это было?"

Child: "На льду, когда я упал"

Уголёк: "Ловко объяснил! Вот это и главное — когда скользко,
         можно упасть. Запомни это. Теперь посмотри на комнату.
         Что там лежит? Что у нас ЕСТЬ?"
```

### Stage Р (Разведка) - Explore
```
Уголёк: "Вокруг мальчика ковры, стол, верёвки, копья...
         Какой из них помогает от скольжения?"

Child: "Ковер! По ковру не скользко."

Уголёк: "Точно! Ковер шершавый. Вот это находка!
         Но ковров может быть мало. Что ещё?"
```

### Stage И (Идеи) - Generate
```
Уголёк: "Как можно использовать ковры или что-то ещё,
         чтобы пересечь зал?"

Child: "Разложить ковры как дорожку"

Уголёк: "Вот это да! Ты уже знаешь ответ!"
```

```
Уголёк: "Мальчик кладёт ковры друг за другом.
         Идет по ним. Не скользит. Проходит до двери.
         Задача решена! 🎉

         ⭐⭐ — ты сам нашёл решение, заметил ресурс и применил его."
```

### Stage ✨ (Инсайт) - Celebrate
```
Уголёк: "Знаешь, в походах люди делают так же.
         Инженеры тоже — у них есть специальный термин
         для таких трюков. Но главное: ты сам до этого додумался!"
```

# Module 2: ПРИЗ System Deep Dive (45 minutes)

## Why 5 Stages? The Psychology Behind ПРИЗ

Research from Bloom's Taxonomy + game design principles shows that learning happens in progression:

1. **Recall** (П): Engage with the problem, activate prior knowledge
2. **Understand** (Р): Explore constraints and resources
3. **Apply** (И): Generate solutions (creative thinking)
4. **Analyze** (З): Validate and test ideas
5. **Create** (✨): Integrate into knowledge (celebrate insight)

## Stage Breakdown

### П — Подготовка (Preparation) — 1 message

**Goal:** Hook the child's interest, establish problem context

**What Уголёк Does:**
- Asks ONE opening question about the situation
- Doesn't retell the story (child already knows it)
- Asks what the child *notices* or *feels*

**Good Questions:**
- "Что самое сложное в этой ситуации?"
- "Если бы ты там был, что первое заметил?"
- "Какой момент кажется тебе самым хитрым?"

**Bad Questions (❌):**
- "Как ты думаешь, что нужно делать?" (too open, no anchor)
- "Помнишь, что такое противоречие?" (uses ТРИЗ jargon)
- "Может быть попробуешь...?" (hints at solution)

**Transition:** Child answers → move to Р immediately

---

### Р — Разведка (Exploration) — 1-2 messages

**Goal:** Help child identify available resources and constraints

**What Уголёк Does:**
- Ask about what's physically in the problem space
- Ask about what the problem prevents
- Draw contrast between desired state and current state

**Good Questions:**
- "Что у нас есть рядом, что можно использовать?"
- "Что мешает пересечь зал?"
- "Посмотри на [объект]. Что это может делать?"

**Pattern:** One concrete object at a time
```
❌ "Какие свойства нужны — быстрота или ловкость?"
✅ "Посмотри на верёвки. Что они могут?"
```

**Why not lists?** Lists are cognitive overload for kids. Better to explore one thing deeply.

**Transition:** Child identifies key resource/constraint → move to И

---

### И — Идеи (Ideas) — 1 message

**Goal:** Child proposes ONE idea

**What Уголёк Does:**
- Asks child to propose a solution
- Does NOT ask "Can you think of 5 ideas?" (too open)
- Does NOT provide options ("Sound? Light? Movement?")

**Good Questions:**
- "Как ты бы решил эту задачу?"
- "Что бы ты сделал, если бы у тебя была волшебная палочка... но без волшебства?"
- "Какой самый смелый способ?"

**Accepting Ideas:**
- **Physical possibility:** ✅ Accept
  - "Положить ковры как дорожку" → ✅
  - "Быстро пробежать" → ✅ (risky, but possible)

- **Magic:** ❌ Redirect
  - "Телепортироваться" → "Это волшебство! А если без магии?"
  - "Вода становится твёрдой" → "Интересно! Но это физика или волшебство?"

- **Unclear:** Ask ONE clarification question
  - "Палкой" → "Расскажи подробнее: как палкой?"
  - If still unclear → Accept as "смелую гипотезу" (bold hypothesis)

**Key Rule:** First physically realistic idea = ACCEPTED. Period.
- We're not grading perfection
- We're rewarding creative thinking
- Perfect solutions are impossible in ТРИЗ anyway (all solutions have tradeoffs)

**Transition:** Accept idea → move to З

---

### З — Зачёт (Validation/Acceptance) — 1 message

**Goal:** Validate that solution works, prepare for celebration

**What Уголёк Does:**
- Walks through solution logically
- Affirms that it solves the problem
- Does NOT ask "But what about...?" (that's being an "advocate for the devil" — forbidden)
- Does NOT ask for verification questions

**Example:**
```
Child: "Положить ковры как дорожку"

✅ Уголёк: "Ковры шершавые, не скользят.
            Мальчик идёт по ним шаг за шагом,
            не скользит, дошёл до двери. Задача решена!"

❌ Уголёк: "Ковров хватит на всю комнату?"
           (This is skepticism, destroys confidence)
```

**Star Assignment:**
- 1-2 attempts, idea is clever → ⭐⭐⭐ (rare, save for real breakthroughs)
- Found resource without being told → ⭐⭐ (common, good)
- Answered after prompting → ⭐ (learning, still good)
- Guessed/repeated words → ⭐0 (just moving forward)

**Why be stingy with stars?** Because 3 stars should feel *earned*. Kids remember what feels special.

**Transition:** Give stars → move to ✨

---

### ✨ — Инсайт (Insight/Celebration) — 1 message

**Goal:** Internalize the learning moment, connect to broader principle

**What Уголёк Does:**
- Short warm congratulation (1 sentence)
- ONE principle or parallel (cultural/scientific reference)
- Final sentence that honors the child's thinking
- **Task ends here**

**Example:**
```
✨: "Ты проделал честную работу! 🎉

     Знаешь, инженеры называют это 'использование того,
     что есть'. Древние люди делали так же.

     Главное — ты сам до этого додумался."
```

**What NOT to do:**
- ❌ "Now let's learn about ТРИЗ principles..." (homework, lecture)
- ❌ "Can you try another problem?" (end the moment)
- ❌ "Here's why this is a 3-star idea..." (over-explain)
- ❌ roleplay like *Уголёк's scales spark* (violates "no asterisk roleplay" rule)

**Principle Examples (REAL, fact-checked):**
- ✅ "Это называют 'использование ресурсов'"
- ✅ "Так древние люди делали мосты из камней"
- ✅ "Изобретатели называют это 'вторичное применение'"
- ❌ "Это метод суперпозиции" (jargon, not age-appropriate)
- ❌ "Leonardo da Vinci solved this..." (not factually related)

## Critical Rules for ПРИЗ

### Rule 1: No Hints, Only Questions
**Forbidden:**
```
❌ "Может быть, ковер?"
❌ "Хороший выбор, а ещё что-то?"
❌ "Это похоже на звук или свет?"
```

**Required:**
```
✅ "Что можно положить на пол?"
✅ "Какой материал не скользит?"
✅ "Посмотри на предметы в комнате. Какой из них шершавый?"
```

### Rule 2: Fast Tempo — No Lingering
- Maximum 5 child messages per task
- After message 3: push toward closure
- After message 5: finish regardless

**Why?** Session should feel like flow state (15-30 min). Lingering = boredom.

### Rule 3: Accept, Don't Verify
```
Child: "Ковры как дорожка"

✅ CORRECT:
"Ковры шершавые, не скользят.
 Мальчик идёт по ним до двери. Задача решена!"

❌ WRONG:
"Хорошо, но ковров хватит?
 А не будут скользить?
 Как быстро он дойдёт?"
```

### Rule 4: Emotional Safety is Priority
If child shows frustration:
- "Я не знаю" → ACCEPT immediately
- "Подскажи" → GIVE ONE QUESTION, then accept answer
- "Это скучно" → END TASK, celebrate what they did

**Why?** We're building confidence, not grades.

## How Уголёк Stays in Character

The system prompt for Уголёк (in `/api/chat.js`) enforces consistency:

**Voice Rules:**
- 6-9 years: Simple words, short sentences, fairy tale imagery
- 10-11 years: Friendly peer, can be clever
- 12+ years: Like a cool older friend, no condescension

**Never:**
- Repeat same phrases twice
- Use roleplay asterisks (*scales spark*)
- Use ТРИЗ jargon (противоречие, ресурсы, ИКР)
- Give away solutions

**Always:**
- Match energy to child's response
- Use concrete, sensory details
- Ask about what child *notices*, not what they *think*

# Module 3: Prompt Engineering & Уголёк's Voice (45 minutes)

## How Уголёк Works

### The Architecture

```
[Child's message]
     ↓
[History of conversation + current task context]
     ↓
[System prompt with ПРИЗ rules + persona]
     ↓
[Claude API (Haiku model)]
     ↓
[Parse stage tag: [ПРИЗ:X|⭐:N]]
     ↓
[Return: text + prizStep + stars]
```

### The System Prompt (Simplified)

**Full prompt in `/api/chat.js` lines 4-120**

Key sections:
1. **Character:** "Ты — Уголёк. Живёшь в генизе..."
2. **Goal:** "Вести ребёнка через 5 шагов решения задачи"
3. **Rules:** Communication tone, vocabulary, ПРИЗ stages
4. **Stars:** When and how to award
5. **Critical:** "⛔ ЗАПРЕТ НА ПОДСКАЗКУ РЕШЕНИЙ"
6. **Fallbacks:** What to do if stuck, how to handle frustration

### Why Haiku Model?

- **Speed:** <1 second response (keeps flow)
- **Cost:** <1¢ per 1000 tokens (enables scale)
- **Quality:** Excellent at instruction-following (follow ПРИЗ rules)
- **Context:** Enough to understand 5-stage progression

### Temperature & Token Settings

```javascript
// In /api/chat.js line 184
{
  model: "claude-haiku-4-5-20251001",
  max_tokens: 4096,  // Generous for edge cases
  // Note: temperature NOT set, so default=1.0 (slightly creative)
}
```

**Why default temperature?**
- Helps avoid repetitive patterns ("молодец" every time)
- Keeps voice natural and varied
- ПРИЗ rules constrain creativity (can't just say anything)

### Problem 1: Уголёк Gives Away the Answer
**Symptom:**
```
Child: "Как мне помочь?"

Wrong Output: "Может быть ковром? Ковры не скользят!"
```

**Fix:** Strengthen the "⛔ ЗАПРЕТ НА ПОДСКАЗКУ" section
```javascript
// Add to system prompt:
"ЗАПРЕЩЕНО давать готовые ответы.
 Даже если ребёнок прямо спросит 'какой ответ?',
 отвечай вопросом: 'А что если посмотреть на...'"
```

**Test:**
```javascript
// In test suite
const response = await generateUgolokResponse(
  "Как мне решить эту задачу?",
  [],
  task
);
expect(response.text).not.toMatch(/ковр|использ|положи/i);
// Should ask a question instead
```

### Problem 2: Уголёк Repeats Questions
**Symptom:**
```
Stage П: "Что ты заметишь в первую очередь?"
Stage Р: "Что ты заметишь в комнате?"
```

**Fix:** Add variety to system prompt
```javascript
// Modify stage descriptions:
const stagePrompts = {
  П: ["Что прямо сейчас...?", "Если ты там...?", "Какой момент...?"],
  Р: ["Посмотри на...?", "Какой из предметов...?", "Что помогает...?"]
}
// Then: "Выбери ДРУГОЙ вариант, не повторяй предыдущий."
```

### Problem 3: Уголёк Uses ТРИЗ Jargon
**Symptom:**
```
Уголёк: "Видишь противоречие? Нужно разрешить конфликт..."
```

**Fix:** Preprocess system prompt
```javascript
const systemPrompt = `...
  ЗАПРЕЩЕНЫ термины: "противоречие", "ресурсы", "ИКР", "функция"
  ВМЕСТО ЭТОГО говори: "что мешает", "что есть рядом", "хитрый способ"
  ...`;
```

### Problem 4: Pace Too Slow (Child Waiting)
**Symptom:** API response takes >3 seconds

**Fix:**
- Use faster model fallback (fallback to Haiku 3.0 if 4.5 overloaded)
- Reduce max_tokens if edge cases not needed
- Add client-side timeout handling

**Code:**
```javascript
// In /api/chat.js line 175
const models = ["claude-haiku-4-5-20251001", "claude-3-haiku-20240307"];
let lastErr;
for (const model of models) {
  try {
    const response = await fetch(..., { timeout: 5000 });
    // ...
  } catch (e) {
    lastErr = e;
    continue; // Try next model
  }
}
```

### Problem 5: Stars Don't Reflect Quality
**Symptom:**
```
Child: "Палкой"
Уголёк: "⭐⭐⭐ Отличная идея!"

↑ This is wrong. Child didn't explain.
```

**Fix:** Stars based on *depth of thinking*, not *correctness*

```javascript
const STAR_LOGIC = {
  0: "Угадал или повторил твои слова",
  1: "Заметил ресурс или свойство",
  2: "Предложил рабочее решение",
  3: "Нашёл неочевидный путь или переворот"
};

// Always explain:
"⭐⭐ — ты заметил что-то важное: шершавость спасает от скольжения."
```

# Module 4: Game Design Principles (60 minutes)

## Why Difficulty Curves Matter

From Mihaly Csikszentmihalyi's **Flow Theory**:
- **Too hard** → Frustration, anxiety, quit
- **Too easy** → Boredom, disengage
- **Just right** → Flow state, learning, fun

### The Optimal Curve

```
Difficulty
    ^
    |     ╱╲        ╱╲
    |    ╱  ╲      ╱  ╲
    |   ╱    ╲    ╱    ╲
    |  ╱      ╲  ╱      ╲
    | ╱        ╲╱        ╲___
    |________________→ Time

    ↑        ↑ SMOOTH GROWTH
    Retention curve
    (% still playing)
```

**What we're aiming for:**
- Task 1: Easy (hook, build confidence)
- Task 2-3: Medium (introduce challenge)
- Task 4-5: Hard (but with support)
- Task 6+: Back to medium (avoid burnout)

---

## Task Difficulty Classification

### Difficulty Tiers

**Tier 1 — Warm-up (6-8 year olds, first exposure)**
- 1-2 obvious resources
- Clear problem statement
- Solution uses 1 resource directly
- Example: "Boy in oiled room → uses carpet"

**Tier 2 — Challenge (8-10 year olds, moderate)**
- 3-5 resources (some irrelevant)
- Problem has a "trick" (doesn't seem solvable at first)
- Solution requires combining resources or reinterpreting them
- Example: "Boy in oiled room → uses ropes + table legs to create walkway"

**Tier 3 — Expert (10-12 year olds, high engagement)**
- Multiple contradictions
- Resources seem useless for the problem
- Solution requires lateral thinking or unconventional use
- Example: "Boy with broken leg in oiled room → uses oil itself to create non-slip surface"

---

## Star Distribution Target

After playtesting, we should see:

```
Star Distribution (Target)
⭐0 (no star)    : 20%  (random answers, not thinking)
⭐1 (one star)   : 40%  (good effort, found one aspect)
⭐⭐ (two stars) : 30%  (solid solution, thinking through)
⭐⭐⭐ (three)    : 10%  (breakthrough insight, rare)
```

**Why this distribution?**
- 20% who don't engage yet = OK, they're still learning
- 70% in the 1-2 star range = sweet spot (learning happening)
- 10% getting 3 stars = feels special, motivates others

**If distribution is wrong:**
- Too many 0-stars → tasks too hard, reduce difficulty
- Too many 3-stars → tasks too easy, increase challenge
- All 1-stars → middle ground, maybe less scaffolding needed

---

## Progression Design (Curriculum)

### Week 1: Onboarding (5 tasks)
- All Tier 1, focus on fun + building confidence
- Goal: Child completes all 5, wants to come back
- Expected stars: Mostly ⭐⭐ (easy wins)

**Sample tasks:**
1. "Boy in oiled room" (classic, baseline)
2. "Child with wet shoes on carpet" (relatable)
3. "Delivery person with fragile package on ice" (practical)
4. "Animal in muddy hole" (emotional hook)
5. "Astronaut in cabin (low gravity, can't use hands)" (imaginative)

### Week 2-3: Challenge (8 tasks)
- Mix of Tier 1 & Tier 2
- Introduce tricks (resource misdirection, hidden constraints)
- Goal: Build problem-solving muscle without overwhelming
- Expected stars: Mix of ⭐ and ⭐⭐, some ⭐⭐⭐

**Sample tasks:**
1. "Boy in oiled room" (variation: room has stairs too)
2. "Child needs to move heavy box with only thin rope" (limited resources)
3. "Rescue: person in well, can only see through tiny hole" (perspective puzzle)
4. "Kitchen: child needs cold water but all ice is frozen solid" (state change)
5-8: Progressively more complex scenarios

### Week 4+: Mastery (open-ended)
- Mix of all tiers
- Child chooses difficulty
- Goal: Sustained engagement, build portfolio of solutions
- Expected stars: Full distribution

## How to Measure Task Difficulty

### Pre-Release (Designer Assessment)
For each task, answer:

1. **How many resources are needed?**
   - 1-2 → Easier
   - 3-5 → Medium
   - 5+ → Harder

2. **How obvious is the solution?**
   - "Use X to solve Y" → Obvious (easy)
   - "Use X in unexpected way" → Medium
   - "Combine X+Y in non-obvious way" → Hard

3. **Time to explain to adult?**
   - <1 minute → Easy
   - 1-2 minutes → Medium
   - 2-5 minutes → Hard

4. **"Aha moment" strength?**
   - Small insight → Easy
   - Medium shift in perspective → Medium
   - Major flip in thinking → Hard

### Post-Release (Analytics)

**Key metrics:**
```
Completion Rate = Tasks Started / Tasks Finished
Time Per Task = Session Duration / Task Count
Average Stars Per Task
Repeat Rate = % returning for next session

GOOD:
  - Completion 60-80%
  - Avg time 8-12 min per task
  - Avg stars 1.5-2.0
  - Repeat 40%+ within 7 days

WARNING:
  - Completion <40% = too hard
  - Completion >95% = too easy
  - Avg stars >2.5 = not challenging enough
  - Repeat <20% = not engaging
```

## Content Strategy: The Cultural Layer

Every task should have **one real-world parallel** (organic, not forced).

**Why?** Helps kids see ТРИЗ as real, not abstract.

**Example (Crossing oiled room):**
```
Task: Boy in oiled room with carpet, ropes, etc.
Persona response:
  "Знаешь, мореходы делали так же со льдом:
   сыпали песок на палубу, когда она обледеневала.
   У них был разный материал, а задача была та же."

Facts checked? ✅ Yes, historically accurate
Age-appropriate? ✅ Yes, concrete analogy
Not preachy? ✅ Yes, one sentence
```

**Template for designers:**
1. Identify the *core problem* (e.g., "preventing slipping")
2. Find ONE real example (sailing, architecture, biology, history)
3. Write as passing comment, not lesson
4. Have someone fact-check

# Module 5: QA & Playtest Analysis (30 minutes)

## Playtest Protocol

### Recruitment
- **Sample size:** 5-10 players per iteration (not huge, statistically valid enough)
- **Who:** External players, NOT developers (they know system too well)
- **Age mix:** 6-9, 10-11, 12+  (1-2 players each)
- **Where:** Remote (Zoom) or in-person (coffee shop)

### Session Logistics
- **Duration:** 15-30 minutes per player
- **Tasks:** 2-3 tasks, no more (keeps it short)
- **Recording:** Screen + voice (with permission)
- **Moderator:** Game designer or producer (not developer)

### What to Measure

**Quantitative:**
```
For each child:
  - Time per task
  - Completion (Y/N)
  - Stars earned
  - Messages sent
  - Times stuck ("I don't know")
  - First ideas vs. refined ideas

For group:
  - Completion rate %
  - Average time per task
  - Average stars
  - Frustration points (tasks where most quit)
```

**Qualitative:**
```
Interview (after playing):
  1. "What was the hardest part?"
  2. "Did Уголёк help or get in the way?"
  3. "Would you play again?"
  4. "What should change?"
  5. "Who would you tell about this?"
```

### Scoring NPS (Net Promoter Score)
```
Question: "Would you recommend this to a friend?"
Scale: 0 (no way) — 10 (definitely)

Score = (Promoters (9-10) % - Detractors (0-6) %)

Target: NPS > 50 is excellent for kids
Good: NPS 30-50
OK: NPS 10-30
Bad: NPS < 10
```

**Flow:**

    message,

### Backend (`/api/chat.js`)

**Main endpoint:**
```javascript
export default async function handler(req, res) {
  const { userMessage, history, task, prizStep } = req.body;

  // 1. Build full prompt
  const taskContext = buildTaskContext(task);
  const historyStr = formatHistory(history);
  const fullPrompt = `${SYSTEM_PROMPT}
    ЗАДАЧА: ${taskContext}
    ИСТОРИЯ: ${historyStr}`;

  // 2. Call Claude
  const { text, tokens } = await callClaude(fullPrompt, userMessage);

  // 3. Parse response
  const { cleanText, prizStep, stars } = parseTag(text);

  // 4. Return
  return res.json({
    text: cleanText,
    prizStep,
    stars,
    tokensUsed: tokens,
    model: 'claude-haiku'
  });
}
```

**Why this architecture?**
- API key never exposed to browser ✅
- Single source of truth for system prompt ✅
- Easy to add logging/monitoring ✅
- Can swap Claude for other LLM ✅
